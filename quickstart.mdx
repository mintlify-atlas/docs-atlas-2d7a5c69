This guide walks you through the minimal steps to get Pulse working: install the packages, capture `URLSession` traffic, store a log message, and open the console.

## Install Pulse

Add the Pulse package to your Xcode project via Swift Package Manager using the URL:

```
https://github.com/kean/Pulse
```

Add the **Pulse** and **PulseUI** libraries to your app target. See [Installation](/installation) for full details, including CocoaPods and binary options.

## Capture network traffic

<Tabs>
  <Tab title="URLSessionProxy (recommended)">
    `URLSessionProxy` is a drop-in wrapper around `URLSession`. Use it in debug builds and fall back to a plain `URLSession` in production.

    ```swift
    import Pulse

    #if DEBUG
    let session: URLSessionProtocol = URLSessionProxy(configuration: .default)
    #else
    let session: URLSessionProtocol = URLSession(configuration: .default)
    #endif
    ```

    Because both `URLSessionProxy` and `URLSession` conform to `URLSessionProtocol`, you can type your networking layer against the protocol and swap the implementation at compile time without any other changes.

    <Tip>
      `URLSessionProxy` also enables response mocking when connected to Pulse Pro. See the [network logging guide](/network-logging/capturing-requests) for advanced configuration.
    </Tip>
  </Tab>
  <Tab title="PulseProxy (quickest)">
    **PulseProxy** uses swizzling to capture all network traffic from your app automatically — no changes to your networking code required.

    First, add the **PulseProxy** library to your app target (debug scheme only).

    Then call `enableProxy()` at app startup:

    ```swift
    import PulseProxy

    #if DEBUG
    NetworkLogger.enableProxy()
    #endif
    ```

    <Warning>
      PulseProxy uses swizzling and private APIs. Never include it in App Store or production builds. Wrap all usage in `#if DEBUG`.
    </Warning>
  </Tab>
</Tabs>

## Store log messages

Use `LoggerStore.shared` to record messages from anywhere in your app:

```swift
import Pulse

LoggerStore.shared.storeMessage(
    label: "auth",
    level: .debug,
    message: "Will login user",
    metadata: ["userId": .string("uid-1")]
)
```

<Tip>
  If your app already uses [SwiftLog](https://github.com/apple/swift-log), you can use Pulse as a log backend via `PersistentLogHandler` from the [PulseLogHandler](https://github.com/kean/PulseLogHandler) package instead.
</Tip>

## View logs on-device

Add `ConsoleView` from **PulseUI** to your app. The simplest approach is to push it from a debug menu or settings screen:

```swift
import PulseUI

NavigationLink(destination: ConsoleView()) {
    Text("Console")
}
```

Or present it modally:

```swift
import PulseUI

.sheet(isPresented: $isConsolePresented) {
    ConsoleView()
}
```

## Enable remote logging

To stream logs in real time to the [Pulse Pro](https://pulselogger.com) app on macOS or iOS, enable automatic connection at app startup:

```swift
import Pulse

#if DEBUG
RemoteLogger.shared.isAutomaticConnectionEnabled = true
#endif
```

You also need to add two entries to your app's `Info.plist` to allow local network access:

```xml
<key>NSLocalNetworkUsageDescription</key>
<string>Network usage required only for development purposes</string>
<key>NSBonjourServices</key>
<array>
  <string>_pulse._tcp</string>
</array>
```

<Note>
  Pulse will not show any network permission prompts until remote logging is actually enabled. The `Info.plist` entries are required by the OS but the user is only prompted when the feature is used.
</Note>

Once connected, open Pulse Pro and select your device in the sidebar. The connection will be re-established automatically on subsequent launches.
